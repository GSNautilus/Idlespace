# Ship Card Styling Guide

Short reference for tweaking the ship / pirate card design in [starmap.html](starmap.html).
Other card types (star, planet, admiral, building, leader) still use the legacy renderer — this guide is ship-only for now.

All ship-card CSS is scoped under **`.card-preview.is-ship`** and lives in the big `<style>` block in `starmap.html`. Markup is generated by **`buildShipCardHtml(card)`** further down in the same file.

## Anatomy (top to bottom)

```
.card-preview.is-ship             ← outer container, drop-shadow glow
├── .bg                           ← gunmetal gradient + brushed streaks + glint
├── .frame-corners                ← 4 L-shaped corner brackets
└── .c                            ← content wrapper (position: absolute, inset: 0)
    ├── .cost                     ← cost chip, absolutely positioned top-right
    ├── .hdr                      ← header plate with ship name + underline
    │   └── .hdr-name
    ├── .img                      ← image frame (::before = corner ticks, ::after = vignette)
    │   └── <img>
    ├── .subtype                  ← "common · Ship" sub-label
    ├── .row                      ← defense stats (shields / armor / speed)
    │   └── .stat × N             ← each: <svg class="ico"> + .sv value
    ├── .weap                     ← ARMAMENTS block (label in ::before)
    │   └── .wr × N               ← each: <svg class="ico"> + .sv + .rb (S/M/L badge)
    ├── .row.u                    ← utility stats (sensor / pd / stealth)
    ├── .special                  ← gradient pill (only if card.special is set)
    ├── .flv                      ← flavor text (italic, hairline above)
    └── .wm                       ← ship-silhouette watermark, absolute bottom
```

## Key levers

### Colors — rarity drives everything
Every rarity-tinted element (`.bg`, border, corner brackets, subtype, stat icons, flavor, weapon-block divider, etc.) reads from the CSS variable **`--rc`**. It's set per rarity:

```css
.card-preview.is-ship.common    { --rc: var(--rarity-common); }
.card-preview.is-ship.uncommon  { --rc: var(--rarity-uncommon); }
.card-preview.is-ship.rare      { --rc: var(--rarity-rare); }
.card-preview.is-ship.epic      { --rc: var(--rarity-epic); }
.card-preview.is-ship.legendary { --rc: var(--rarity-legendary); }
```

Rarity colors themselves live in [ui/colors_and_type.css](ui/colors_and_type.css) as `--rarity-common` through `--rarity-legendary`. Change them there and every card updates.

### Card size
`.card-preview` (not `.is-ship`) sets `width: 220px; height: 320px`. Lots of JS (slot fitting, scaling) assumes this — don't change without a search.

### Title (`.hdr`)
Move or restyle the ship name:
- `.hdr { padding }` — horizontal + vertical room around the name
- `.hdr .hdr-name { font-size }` — name size
- `.hdr::after` — the glowing underline; `bottom` controls its distance from the name
- To center the title, add `text-align: center` to `.hdr`

### Cost chip (`.cost`)
- `.cost { top, right }` — position in top-right
- `.cost { font-size, color }` — gold text (`var(--res-credits)`)
- To change the icon, edit `RES_ICON_BY_KEY` (maps resource keys → sprite ids)

### Image frame (`.img`)
- `height` — image area height (currently 128px)
- `margin` — inset from card edges
- `box-shadow` — 1px rarity-tinted inner border, inner vignette, outer drop
- `.img::before` — the rarity-colored corner tick marks (8 gradient layers)
- `.img::after` — the scanline + central vignette overlay
- `.img img { object-position }` — crop the portrait (`center 42%` biases toward upper half)

### Sub-label (`.subtype`)
The "common · ship" text under the image. Built in JS as:
```js
const subLabel = `${card.rarity} · ${card.category === "pirate" ? "Pirate" : "Ship"}`;
```

### Stats (`.row` / `.stat`)
- `.row { padding, gap }` — spacing between chips
- `.stat` — single chip (icon + value)
- `.stat .ico.shield/.armor/.sensor` — override per-icon color; default is `var(--fg-muted)`
- `.sv` — the value itself (white, 13px, tabular-nums)

### Weapons (`.weap` / `.wr`)
- `.weap::before { content: 'ARMAMENTS' }` — change the floating section label here
- `.wr` — one weapon row. Left border is `var(--wr-c, var(--rc))`; the renderer sets `--wr-c` inline per weapon (`var(--wpn-energy)`, `var(--wpn-kinetic)`, `var(--wpn-missile)`)
- `.rb.S`, `.rb.M`, `.rb.L` — range-badge colors; they pull from `--range-short/medium/long` in `colors_and_type.css`

### Special / flavor
- `.special` — gradient pill; fades rarity into the gunmetal background
- `.special .sn` — the "Special" label at the top
- `.flv` — italic flavor; `.flv::before` is the short hairline above it

### Watermark (`.wm`)
Faint ship silhouette behind the flavor. Uses the `#i-solo-ship` sprite symbol. Change `width/height` to resize, `opacity` to make it more or less visible, or swap the `<use href>` target in the renderer for a different sprite.

## Where to edit

| What | Where |
|---|---|
| Any CSS above | `<style>` block near top of [starmap.html](starmap.html), scoped under `.card-preview.is-ship` |
| Markup / what data goes into each slot | `buildShipCardHtml(card)` function in [starmap.html](starmap.html) |
| Icon sprite (add a new icon) | [ui/icons.html](ui/icons.html) — add a `<symbol id="i-...">` |
| Rarity palette | [ui/colors_and_type.css](ui/colors_and_type.css) — `--rarity-*` |
| Resource / weapon / range palette | [ui/colors_and_type.css](ui/colors_and_type.css) — `--res-*`, `--wpn-*`, `--range-*` |
| Shared "metal" surface + `.ico` helper | [ui/chrome.css](ui/chrome.css) |

## Common pitfall

Adding `backdrop-filter` anywhere inside the card (tried once for a terminal-style badge) will silently hide every sibling below the image — it interacts badly with the card's outer `drop-shadow` filter and the `mix-blend-mode: overlay` on `.bg::before`. Use `rgba(0,0,0,0.7)` or similar when you need a frosted look.
